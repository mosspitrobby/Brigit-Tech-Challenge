# Bridgit competency test

This is a NestJS web service that provides the backend APIs for the competency test.

### Table of contents

- [Setting up](#setting-up)
- [Swagger documentation](#swagger-documentation)
- [Folder structure](#folder-structure)
- [Deploying](#deploying)

## Setting up

If you do not have NodeJS installed you will need Node 18.

Tip:  You can use `nvm` and prefix commands with `nvm exec` to automatically preference the appropriate NodeJS if you are using `Node Version Manager`.

```bash
$ git clone git@bitbucket.org:bridgit-tech/tech-test-api.git
$ cd tech-test-api
$ cp .env.localdev .env
$ npm install --include=dev
$ npx nest start
# nvm exec npm install --include=dev
# nvm exec npx nest start
```

## Swagger documentation

You will need Swagger as a reference when implementing the API in your frontend.  API documentation is generated by Swagger and accessible at `/swagger` when running the web server, eg:

```bash
# view API docs at http://localhost:9300/swagger
$ npx nest start 
```

## Folder structure

| Folder                | Description                                                               |
| --------------------- | ------------------------------------------------------------------------- |
| src/api               | Controllers and endpoints for the API and DTOs for request/response data  |
| src/common            | Constants, Nest utilities                                                 |

## Deploying

This project should deploy upon any NodeJS web hosting service.

Tip: Remember to set the `CORS_ORIGINS` environment variable with your frontend server address to allow communication between them:

```bash
CORS_ORIGINS=https://my-app.netlify.app
```
